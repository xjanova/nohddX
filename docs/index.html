ปัญหาแอดมินที่ NoHddX แก้ได้

ห้องแล็บคอม โรงเรียน อินเทอร์เน็ตคาเฟ่ คีออสก์ POS ออฟฟิศ — ทุกที่ที่มีเครื่องเหมือนกันจำนวนมาก มีปัญหาเดียวกันหมด

เดิม: บริหารทีละเครื่อง

  • ลง Windows ใหม่ทุก 6 เดือนเพราะไวรัส / ผู้ใช้ลงโปรแกรมมั่ว
  • SSD พังต้องซื้อใหม่ + ลง OS ใหม่
  • อัปเดตซอฟต์แวร์ต้องเดินไปลงทีละเครื่อง
  • เครื่อง 50 ตัวมี 50 สถานะ ไม่มีใครรู้ว่าตัวไหนพร้อมใช้
  • ผู้ใช้ลบไฟล์ระบบเล่น — เครื่องเจ๊ง

NoHddX: ภาพเดียวคุมทั้งฟลีต

  • แก้ image กลางครั้งเดียว เครื่องทั้งหมดอัปเดตพร้อมกัน
  • เครื่องไคลเอนต์ไม่มีดิสก์ → ไม่มีอะไรพัง ไม่มีอะไรหาย
  • รีบูตเครื่อง = รีเซ็ตกลับสภาพเดิม (Copy-on-Write overlay)
  • Dashboard เดียวเห็นสถานะทั้งฟลีตแบบเรียลไทม์
  • Wake-on-LAN เปิด/ปิดทั้งห้องจากที่นั่งแอดมิน
หลักการทำงาน

บูตจากศูนย์สู่หน้าจอ Windows ใน 5 ขั้น

ทุกครั้งที่กดเปิดเครื่อง flow เดิมจะเกิดขึ้นอัตโนมัติ — ไม่มี HDD ไม่มี SSD ไม่มี USB ติดเครื่องก็ยังบูตได้ ขอแค่มี Network Card รองรับ PXE (มาตรฐานทุกเมนบอร์ดยุคใหม่)

1
PXE / DHCP
เมนบอร์ดขอ IP และถาม "บูตจากไหน" — NoHddX DHCP-Proxy ตอบกลับ
2
TFTP → iPXE
เครื่องดาวน์โหลด iPXE binary จาก TFTP ของเซิร์ฟเวอร์
3
HTTP script
iPXE เรียก /api/boot/{mac}.ipxe เพื่อรับ script ส่วนตัวของเครื่องนั้น
4
iSCSI sanboot
เครื่อง mount LUN ของตัวเองจาก iSCSI target — ใช้เหมือนดิสก์จริง
5
Boot OS
Windows/Linux โหลด bootloader จาก iSCSI แล้วบูตขึ้นปกติ

🧠 Logic หลัก: Copy-on-Write overlay

เครื่องลูกข่าย หลายเครื่อง ใช้ image base เดียวกัน แต่เขียนทับลงไปได้ — เพราะทุก write จะถูก redirect ไปยัง overlay VHD เฉพาะตัวของเครื่อง ๆ นั้น ทำให้:

# 1 image base = 30 GB ใช้ร่วมกัน 100 เครื่อง storage/bases/windows11-lab.vhd // 30 GB, read-only # แต่ละเครื่องมี overlay ของตัวเอง (เก็บเฉพาะ block ที่เปลี่ยน) storage/overlays/AA-BB-CC-DD-EE-01.vhd // ~500 MB storage/overlays/AA-BB-CC-DD-EE-02.vhd // ~500 MB # แอดมินคลิก "Reset" → ลบ overlay เครื่องนั้น # รอบบูตหน้าเครื่องกลับสะอาดเอี่ยมเหมือนวันแรก

ทุกอย่างที่ Diskless platform ระดับ enterprise ควรมี

💾

iSCSI Target ในตัว

พูด RFC 3720 จริง — รองรับ CHAP MD5, HeaderDigest/DataDigest (CRC32C), R2T, multi-PDU writes บูต Windows / Linux ได้

🌳

Copy-on-Write Overlay

Per-client VHD overlay บน base image เดียวกัน — ประหยัดดิสก์ 95%+ รีเซ็ตกลับสภาพต้นได้ทันที

🚀

PXE / iPXE / TFTP

DHCP-Proxy ไม่ชนกับ DHCP เดิมในเครือข่าย, มี TFTP server ในตัว และสร้าง iPXE script ต่อ MAC อัตโนมัติ

🔐

Security & Audit

X-Admin-Api-Key + Bearer token (HMAC-SHA256) ต่อ agent, per-IP rate limit, request-size cap, audit log ทุก mutation

🌐

Raft Cluster (Optional)

หลายโหนดเลือก leader, sync state ระหว่างกัน — ฟลีตใหญ่ ๆ ไม่มี single point of failure

📊

SignalR Live Dashboard

WPF console สำหรับแอดมิน — เห็น client status, boot event, cluster health แบบ push real-time

🔔

Prometheus Metrics

Export metrics + alerts มาตรฐาน เสียบเข้า Grafana / Alertmanager ได้ทันที

⚡

Wake-on-LAN

ปลุกเครื่องทั้งห้องจาก dashboard, schedule เปิด/ปิดอัตโนมัติ ลด power cost

🛠️

USB / ISO Builder

CLI tool สร้าง bootable USB image ให้เครื่องที่ NIC ไม่รองรับ PXE — เสียบ USB แล้วบูตได้เลย

Use cases ที่ NoHddX เปลี่ยนเกมได้

🏫

โรงเรียน / มหาวิทยาลัย

ห้องคอม 30-60 เครื่อง รีเซ็ตทุกชั่วโมงเรียน อัปเดตโปรแกรมสอนจากตัวเดียว

🎮

อินเทอร์เน็ตคาเฟ่ / เกมมิ่ง

เครื่องลูกค้า 50-200 เครื่อง — ลูกค้าจบ session, overlay reset, เครื่องสะอาดสำหรับคนถัดไป

🏦

คีออสก์ / POS / ATM-like

🏢

ออฟฟิศ / Call center

พนักงานหมุนเวียน — ทุก seat บูต image เดียวกัน OS อัปเกรดทั้งบริษัทพร้อมกันคืนเดียว

เริ่มต้นใช้งาน

Quickstart 3 ขั้น

1. รัน server

.\scripts\start-server.ps1 # Server ฟังที่ http://localhost:8080 # Swagger UI: http://localhost:8080/swagger

2. สร้าง USB boot stick (สำหรับเครื่องที่ NIC ไม่รองรับ PXE)

.\scripts\build-usb.ps1 -ServerUrl http://192.168.1.10:8080 # จะได้ไฟล์ dist/nohddx-boot.img — flash ลง USB ด้วย Rufus / dd

3. บูตเครื่องเป้าหมาย

# 1. เสียบ USB / เปิด PXE boot ใน BIOS # 2. เครื่องโหลด iPXE → ขอ DHCP → chainload # http://<server>/api/boot/<mac>.ipxe # 3. NoHddX ตอบ "sanboot iscsi:..." → เครื่องบูตจาก iSCSI # 4. ดู dashboard เห็น client ออนไลน์ — assign image จาก WPF console

FAQ

เครื่องลูกข่ายต้องมีสเปคขั้นต่ำเท่าไหร่?
แค่มี NIC ที่รองรับ PXE (มาตรฐานทุกเมนบอร์ดยุคใหม่) ไม่ต้องมี HDD / SSD เลยก็ได้ ถ้า NIC เก่าไม่รองรับ PXE ใช้ USB ที่ build จาก build-usb.ps1 เสียบบูตแทนได้
เน็ตเวิร์กต้องเร็วแค่ไหน?
Gigabit Ethernet ทั้งห้องคือต่ำสุดที่แนะนำ สำหรับ 30-50 เครื่องพร้อมกัน ถ้ามากกว่านั้นแนะนำ 10G uplink ที่ฝั่งเซิร์ฟเวอร์ และ managed switch ที่มี IGMP snooping
บูต Windows ได้จริงไหม? ลิขสิทธิ์ Windows มีปัญหาหรือเปล่า?
iSCSI target ของเราพูด RFC 3720 เพียงพอที่ Windows 10/11 จะบูตได้จาก sanboot ปกติ — ส่วนเรื่องลิขสิทธิ์ Windows เป็นความรับผิดชอบของลูกค้า (Microsoft มี VDA / RDS license สำหรับ scenario นี้)
ถ้าเซิร์ฟเวอร์ดับ เครื่องลูกข่ายทำงานต่อได้ไหม?
ไม่ได้ — เพราะดิสก์อยู่บนเซิร์ฟเวอร์ ดังนั้นแพลน Enterprise จึงตั้ง Raft cluster หลายโหนดเพื่อ failover อัตโนมัติ และเก็บ image ไว้บน storage redundant (RAID / Ceph)
ต่างจาก FOG Project / CCBoot / Ardence ยังไง?
NoHddX เขียนใหม่บน .NET 8 — modular, REST API จริง, มี SignalR live dashboard, มี cluster mode ในตัว และที่สำคัญ ซอร์สโค้ดเปิด ไม่ต้องจ่ายต่อ seat ไม่ต้องผูกกับเซิร์ฟเวอร์ของ vendor
มี Linux client ไหม?
มี — โปรเจค NohddX.Agent เป็น mini-OS Linux ที่ตอบ UDP discovery, ลงทะเบียนตัวเอง, และให้แอดมินเลือก mode: persistent install / diskless USB+net / network boot
